【知识点分类】
本章中的知识点需划分为以下五类（必须严格使用这些名称）：

1. 输出格式必须是一个 JSON 数组。
2. 每个元素代表一张知识闪卡，包含以下字段：

- “知识点名称”为具体概念、实验或物理现象名称
- 不使用括号，不添加任何多余文字

【配图信息的添加规则】
- 仅在“实验类”和“物理现象类”，以及确实需要图像辅助理解的知识点中添加
- 配图信息必须放在 back 内容的最后
- 与前文用一行水平线 `---` 分隔
- 采用以下固定格式书写：

![图片描述]

【图片描述固定句式（必须严格遵守）】
- 图片描述必须以“图片类型”开头，且只出现一次图片类型标识（不要重复写两次类型）
- 统一句式为：

<图片类型>：一张表示/描述<主题>的图片，具体要求：<构图与关键细节>

其中：
- <图片类型> 只能从以下中选择其一：示意图 / 实物照片 / 操作步骤图 / 对比图
- “一张表示/描述<主题>的图片”必须明确说明画的是什么（先主题，后细节）
- “具体要求”用于列出必要细节，要求紧凑、可直接用于生成图片

【图片描述内容要求】
图片描述必须简洁但信息完整，需覆盖以下要点（按自然语言合并表达，不要分条）：
- 场景主体（出现的物体/人物及其相对位置）
- 关键关系或关键动作（对齐、方向、参照物、对应关系、起点终点、往返路径等）
- 必要的观察视角（如正上方、侧视；若无要求可不写）
- 必须强调的关键细节或易错点（例如“参照物是地面而非屏幕”“路程与时间必须对应同一过程”等）
- 明确不涉及的内容（可用一句话说明，如“不涉及频率测量/不涉及多次反射/不涉及分贝测量过程”等）
